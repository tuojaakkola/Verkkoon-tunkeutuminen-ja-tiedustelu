Karvinen 2025: Wireshark - Getting Started
- Asennus: sudo apt-get install wireshark
- Lisää itsesi ryhmään sudo adduser $USER wireshark 
- Kaappauksen aloitus: Valitse verkkokortti tai "any" ja paina sinistä
- Suodattimet: Kirjoita yläpalkkiin esim. dns, http tai tls löytääksesi tiettyä liikennettä.
- Analysointi: Käytä Statistics-valikkoa yleiskuvaan ja Follow TCP Stream -toimintoa keskustelujen lukemiseen.
- Tallennus: Tallenna liikenne .pcap-muotoon myöhempää tarkastelua varten.

Karvinen 2025: Network Interface Names on Linux
- Verkkoliitäntä on ohjelmallinen tai fyysinen "verkkokortti" esim. Wi-Fi tai Ethernet
- Nimen alku kertoo tyypin: en (langallinen), wl (langaton) ja lo (loopback/localhost).
- Nykyiset nimet (esim. enp1s0) perustuvat laitteen sijaintiin koneessa, jotta ne pysyvät samoina uudelleenkäynnistyksen jälkeen.
- Omia liitäntöjä ja reitityksiä voi tutkia komennoilla ip a ja ip route

## d)

1. Linkkikerros (Network Access): Linux cooked capture v1
Alin kerros, joka huolehtii datan siirtämisestä fyysisessä verkossa laitteiden välillä. Se käyttää MAC-osoitteita pakettien toimittamiseen paikallisesti.

2. Internet-kerros (Internet Layer): Internet Protocol Version 4
Vastaa pakettien reitityksestä verkkojen välillä. Tässä kohdassa näkyy lähettäjän ja vastaanottajan IP-osoitteet.

3. Kuljetuskerros (Transport Layer): User Datagram Protocol (UDP)
Huolehtii sovellusten välisestä yhteydestä. Tässä kerroksessa määritellään porttinumerot (esim. 443), jotta data löytää oikeaan ohjelmaan.

4. Sovelluskerros (Application Layer): Data (29 bytes)
Ylin kerros, joka sisältää varsinaisen käyttäjälle näkyvän tiedon tai sovelluksen (esim. verkkosivun osa tai nimipalvelukysely) salatussa tai avoimessa muodossa.
